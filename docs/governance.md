# Governance - EventAI

## Risk Register

| # | Риск | P/I | Детект | Защита | Остаточный риск |
|---|---|---|---|---|---|
| 1 | **LLM API недоступен** (429/503, ограничение частоты) | С/В | Мониторинг HTTP-статусов, fail_count | Ротация ключей, пауза 60 с, запасная модель, деградация по уровням | Низкий: CRUD работает, AI деградирует |
| 2 | **Невалидный JSON от LLM** при кластеризации | В/С | Валидатор после каждого вызова | Двухслойная валидация, автоисправления (пропуски - наименьший зал, дубликаты - первое назначение), повтор при ошибках синтаксиса | Низкий: ~3-5% проектов требуют ручной доводки |
| 3 | **Prompt injection** (три вектора: текст пользователя, описания проектов, артефакты) | С/С | Лог вызовов инструментов | Белый список инструментов, валидация аргументов, ограничение длины, артефакты как user message (не system), модерация организатором | Средний: нет автофильтрации |
| 4 | **Утечка персональных данных** во внешний LLM API | Н/В | Аудит отправляемых полей | ФИО и username не отправляются. Из проектов - title/tags/description. Из экспертов - только теги | Средний: нет автоматического PII-детектора |
| 5 | **Потеря сессии** при перезапуске контейнера | С/Н | Мониторинг uptime | PicklePersistence, восстановление из БД по /start | Низкий: теряется только незавершенный диалог |
| 6 | **Перегрузка в день DD** (200 сессий) | С/В | Мониторинг RPS, длина очереди Celery | Celery для асинхронных LLM-операций, кеш Q&A (TTL 1 ч), пакетная обработка уведомлений | Средний: нет ограничения частоты запросов |
| 7 | **Нерелевантные рекомендации** | С/Н | Офлайн-оценка (NDCG/Precision/Recall) | Перепрофилирование, сравнение проектов, откат на пересечение тегов | Средний: нет онлайн-метрик |
| 8 | **Qdrant недоступен** | Н/С | Проверка здоровья в Docker | Автооткат на пересечение тегов (overlap * 20.0) | Низкий |
| 9 | **Злоупотребление ботом** | Н/С | Нет | **Не реализовано.** Ограничение частоты запросов отсутствует | Высокий |
| 10 | **Вредоносный PPTX/PDF** (zip bomb, макросы) | Н/В | Ограничение размера (50 МБ) | Только текстовое извлечение, таймаут 30 с, python-pptx/pymupdf не исполняют макросы | Низкий |
| 11 | **Большой GitHub-репозиторий** | С/С | Мониторинг диска | Поверхностное клонирование, лимит 100 МБ, таймаут 60 с, очистка после парсинга | Низкий |
| 12 | **Потеря оценок экспертов** | Н/В | Проверка записи после каждой оценки | Атомарная запись в PostgreSQL, подтверждение в боте, RPO=0 | Низкий |
| 13 | **Подмена сообщений в канале бот-админка** | Н/С | Авторизация API | Привязка к telegram_id, ответы только от организаторов (белый список), аудит | Низкий |

*P/I = вероятность/влияние: В - высокое, С - среднее, Н - низкое*

## Персональные данные и логирование

### Классификация данных

| Категория | Данные | Уровень |
|---|---|---|
| Идентификаторы | Telegram ID, internal UUID | Псевдонимизированные |
| Персональные | ФИО, Telegram username | 152-ФЗ |
| Профиль интересов | Теги, цели, компания | 152-ФЗ |
| Сырой текст | raw_text диалога, вопросы участников | Может содержать ПД |
| Проектные данные | Названия, описания, теги, артефакты | Общедоступные |
| Оценки экспертов | Баллы по критериям | Конфиденциальные |

### Что отправляется во внешний LLM (OpenRouter)

| Отправляется | Не отправляется |
|---|---|
| Интересы, цели, роль гостя | Telegram username, ФИО, Telegram ID |
| Компания, должность (для бизнеса) | Авторы, контакты проектов |
| Название, теги, описание проектов | Имена, контакты экспертов |
| parsed_pitch, parsed_repo (для агента) | Исходный код целиком, оценки экспертов |
| История диалога (до 20 сообщений) | |

### Логирование

- ФИО и контакты не попадают в логи
- Telegram ID допустим как псевдоним
- LLM-вызовы: model, tokens, задержка, key_id (без содержимого промптов)
- Кластеризация: prompt_hash и input_hash для воспроизводимости

### Локализация

- Хранение в РФ (Yandex Cloud) - 152-ФЗ
- OpenRouter - внешний сервис; отправляемые данные минимизированы (см. таблицу)
- Политика хранения не определена (открытая задача)

## Подтверждение действий

Все критические AI-решения проходят через человека:

| Действие | Кто | Механизм |
|---|---|---|
| Кластеризация | Организатор | Просмотр - перетаскивание - подтверждение |
| Распределение экспертов | Организатор | Просмотр матчинга - корректировки - подтверждение |
| Расписание | Организатор | Просмотр - перетаскивание - экспорт |
| Рассылки/брифинги | Организатор | Предпросмотр - подтверждение отправки |
| Ответ участнику | Организатор | Ручной ввод ответа |
| Профиль гостя | Гость | "Все верно" / "Ввести заново" |
| Оценки | Эксперт | Подтверждение перед записью |
| Выбор слотов | Эксперт | Подтверждение перед отправкой в матчинг |
