# Governance - EventAI Agent

## Risk Register

Риски, относящиеся к агентской части системы.

| # | Риск | P/I | Детект | Защита | Остаточный риск |
|---|---|---|---|---|---|
| 1 | **LLM API недоступен** (429/503, ограничение частоты) | С/В | Мониторинг HTTP-статусов, fail_count | Ротация ключей, пауза 60 с, запасная модель, деградация по уровням | Низкий: агент деградирует, но не ломается |
| 2 | **Prompt injection** (три вектора: текст пользователя, описания проектов, артефакты) | С/С | Лог вызовов инструментов | Белый список инструментов, валидация аргументов, ограничение длины, артефакты как user message (не system) | Средний: нет автофильтрации |
| 3 | **Утечка персональных данных** во внешний LLM API | Н/В | Аудит отправляемых полей | ФИО и username не отправляются; из проектов - название/теги/описание; из экспертов - только теги | Средний: нет автоматического PII-детектора |
| 4 | **Нерелевантные рекомендации** | С/Н | Офлайн-оценка (NDCG/Precision/Recall) | Перепрофилирование, сравнение проектов, откат на пересечение тегов | Средний: нет онлайн-метрик |
| 5 | **Потеря сессии** при перезапуске контейнера | С/Н | Мониторинг uptime | PicklePersistence, восстановление из БД по /start | Низкий: теряется только незавершенный диалог |
| 6 | **Вредоносный PPTX/PDF** (zip bomb, макросы) | Н/В | Ограничение размера (50 МБ) | Только текстовое извлечение, таймаут 30 с, python-pptx/pymupdf не исполняют макросы | Низкий |
| 7 | **Большой GitHub-репозиторий** | С/С | Мониторинг диска | Поверхностное клонирование, лимит 100 МБ, таймаут 60 с, очистка после парсинга | Низкий |
| 8 | **Потеря оценок экспертов** | Н/В | Проверка записи после каждой оценки | Атомарная запись в PostgreSQL, подтверждение в боте, RPO=0 | Низкий |
| 9 | **Злоупотребление ботом** | Н/С | Нет | **Не реализовано.** Ограничение частоты запросов отсутствует | Высокий |

*P/I = вероятность/влияние: В - высокое, С - среднее, Н - низкое*

## Персональные данные

### Что отправляется во внешний LLM (OpenRouter)

| Отправляется | Не отправляется |
|---|---|
| Интересы, цели, роль гостя | Telegram username, ФИО, Telegram ID |
| Компания, должность (для бизнеса) | Авторы, контакты проектов |
| Название, теги, описание проектов | Имена, контакты экспертов |
| parsed_pitch, parsed_repo (для агента) | Исходный код целиком, оценки экспертов |
| История диалога (до 20 сообщений) | |

### Логирование

- ФИО и контакты не попадают в логи
- LLM-вызовы: model, tokens, задержка, key_id (без содержимого промптов)
- Хранение в РФ (Yandex Cloud) - 152-ФЗ
- OpenRouter - внешний сервис; отправляемые данные минимизированы

## Защита от injection

Три вектора атаки: пользовательский текст в Telegram, описания проектов при импорте, содержимое артефактов (презентации, README).

| Мера | Реализация |
|---|---|
| Белый список инструментов | Агент вызывает только описанные функции: show_project, compare_projects, generate_questions, get_followup, get_pipeline, rebuild_profile, ask_organizer |
| Валидация аргументов | Каждый вызов инструмента проверяется по типам и обязательным полям |
| Изоляция контекстов | Персональные данные организатора не попадают в промпты агента |
| Ограничение данных | В LLM - только название/теги/описание проектов; ФИО и контакты не отправляются |
| Артефакты как user message | Содержимое презентаций и репозиториев передается как пользовательское сообщение, не как системное |
